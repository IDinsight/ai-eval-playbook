# Generative AI Evaluation in the Development Sector: A Living Playbook

The use of generative AI (GenAI) tools in low- and middle-income countries is multiplying – from AI-powered math tutors for children to digital advisory tools for farmers. While studies have shown that AI-powered applications can improve human and economic development outcomes (e.g., [Henkel et al., 2024](https://arxiv.org/abs/2402.09809)), others warn of harmful uses (e.g., [Bastani et al., 2024](https://www.google.com/url?q=https://papers.ssrn.com/sol3/papers.cfm?abstract_id%3D4895486\&sa=D\&source=editors\&ust=1770879886855935\&usg=AOvVaw2mkBlmGgvo7SVx26MrPQL2)).

Evaluations can address these challenges and help implementers increase their chances of building impactful GenAI products. Yet there has been little agreement on what this actually means. In the rapidly evolving landscape of GenAI, development sector organizations often find themselves caught between two extremes of evaluation: tech-focused teams optimizing for AI performance (accuracy, latency, cost) but ignoring human outcomes; and development-focused teams jumping straight to Randomized Controlled Trials (RCTs) without verifying if the underlying technology actually works and is used as expected.

These methods are complementary, not mutually exclusive, and this playbook offers a unified framework to bridge the gap. It lays out how to evaluate AI services in the development sector: what evaluation should include, the standard practices implementers should follow. It is organized around a four-level framework:

The four levels form a logical progression. Users are unlikely to stay engaged (Level 2) if the AI system fails to perform (Level 1), and development outcomes are unlikely to improve if users disengage or their feelings, knowledge, and behaviors are harmed.

The central element of this framework is continuous evaluation. Unlike earlier rule-based tools, GenAI performance is highly sensitive to the underlying model(s), training data, task definition, prompts, the context in which it is being applied, how it is configured for a particular use case, and many other parameters. This complexity requires new evaluation methods. Moreover, AI’s core inputs can change far faster than earlier digital technologies. With new foundation models and tools released weekly developers must ensure applications keep behaving as intended over time. Continuous evaluation is essential: it enables rapid iteration, preserves expected behavior, and steadily improves performance and impact.
