# Introduction

{% hint style="info" %}
This playbook is a living playbook. We’ll keep updating this playbook and collaborating more deeply with specialists to co-create shared evaluation tools, refine methodologies, and support their practical use in real-world settings.
{% endhint %}

The use of generative AI (GenAI) tools in low- and middle-income countries is multiplying – from AI-powered math tutors for children to digital advisory tools for farmers. While studies have shown that AI-powered applications can improve human and economic development outcomes (e.g., [Henkel et al., 2024](https://arxiv.org/abs/2402.09809)), others warn of harmful uses (e.g., [Bastani et al., 2024](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=4895486)).

Evaluations can address these challenges and help implementers increase their chances of building impactful GenAI products. **Yet there has been little agreement on what this actually means.** In the rapidly evolving landscape of GenAI, development sector organizations often find themselves caught between two extremes of evaluation: tech-focused teams optimizing for AI performance (accuracy, latency, cost) and user adoption without paying heed to user impacts such as behavior change and other ultimate development outcomes; and development-focused teams focused on ultimate development outcomes, often using Randomized Controlled Trials (RCTs), but with limited emphasis on performance and usage of the underlying technology.

**These foci may seem at odds, but are actually complementary to each other, and not mutually exclusive.** This playbook offers a unified framework to bridge the gap, laying out how to evaluate GenAI services in the development sector: what evaluation should include, and the standard practices implementers should follow. It is organized around a four-level framework:

{% include ".gitbook/includes/4-level-framework-links.md" %}

Even if the boundaries may sometimes be blurry in practice, the four levels form a logical progression. Users are unlikely to stay engaged (Level 2) if the GenAI system fails to perform (Level 1), and development outcomes are unlikely to improve (Level 4) if users disengage or their feelings, knowledge, and behaviors are harmed (Level 3).

The central element of this framework is continuous evaluation. Unlike earlier rule-based tools, GenAI performance is highly sensitive to the underlying model(s), training data, task definition, prompts, the context in which it is being applied, how it is configured for a particular use case and/or user, and many other parameters. This complexity requires new evaluation methods. Moreover, AI’s core inputs can change far faster than earlier digital technologies. With new foundation models and tools released weekly, developers must ensure applications keep behaving as intended over time. Continuous evaluation is essential: it enables rapid iteration, preserves expected behavior, and steadily improves performance and impact.
