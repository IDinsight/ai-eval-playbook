# Minimum Viable Evaluations

| Level 1 - Model evaluation MVE                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| <ul class="contains-task-list"><li><input type="checkbox">At least one <a href="https://app.gitbook.com/s/VDHDXE8axdWQfu0OFCHP/how-to-evaluate/how-is-level-1-evaluation-performed">robust safety/guardrail metric</a> as part of your overall rubric computed on your Golden Dataset</li><li><input type="checkbox">A 30-50 item <a href="https://app.gitbook.com/s/VDHDXE8axdWQfu0OFCHP/how-to-evaluate/how-is-level-1-evaluation-performed">Golden Dataset</a> representing key, diverse user interactions</li><li><input type="checkbox"><a href="https://app.gitbook.com/s/VDHDXE8axdWQfu0OFCHP/how-to-evaluate/how-is-level-1-evaluation-performed">Expert review of AI system</a> responses for inputs in the Golden Dataset, as you iterate on your system configuration</li></ul> |

| Level 2 - Product evaluation MVE                                                                                                                                                                                                                                                   |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <ul class="contains-task-list"><li><input type="checkbox">Baseline user engagement and retention rate tracking</li><li><input type="checkbox">A simple “Helpful/Not Helpful” binary feedback mechanism for users, providing rapid feedback on content or other features.</li></ul> |

| Level 3 - User evaluation MVE                                                                                                                                                                                                                                                                                                                                                                                     |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <ul class="contains-task-list"><li><input type="checkbox">A weekly random sample of 10 conversation logs reviewed by a human expert to identify relevant platform metrics.</li><li><input type="checkbox">Definition of one proximal outcome (e.g., self-reported user confidence) that you can cheaply measure, and that correlates with longer-term impacts (as reported in the academic literature).</li></ul> |

| Level 4 - Impact evaluation MVE                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <ul class="contains-task-list"><li><input type="checkbox">An impact evaluation with counterfactual and enough of a sample size to measure the key outcome(s) of interest, including among sub-populations of interest (e.g. by gender, geography)</li><li><input type="checkbox">Strong version control with either a frozen version or limited number of product versions being tested</li><li><input type="checkbox">Cost data collection</li></ul> |
